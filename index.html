AI 향수 큐레이션 플랫폼 프로젝트 (Scentence) 2025.12~2026.2
참여인원: 5명

대화형 향수 추천 AI 챗봇, 향수 취향 지도, 레이어링 추천 챗봇 등의 기능을 제공하는 향수 큐레이션 플랫폼 서비스 입니다.

역할

데이터
  • 향수 데이터 수집 및 전처리: Parfumo 사이트에서 약 15000여건의 향수의 데이터(향수명, 브랜드, 조향사, 어코드, 노트, 계절등)를 수집하였고 전처리하여 최종적으로 4100여건의 향수데이터를 RDB에 적재하였습니다.
설계
  • 향수 추천 Multi Agent 설계 및 구현: 경로 라우팅, 재질문, 검색, 응답 4가지의 에이전트와 각 에이전트가 사용할 tool을 설계 및 구현하였습니다.
  • 세부 추천 로직 설계: 사용자의 입력에서 성별, 상황, 요청 분위기 등의 정보를 탐색하여 추천에 필요한 정보들의 최소치를 정의하고 이를 통해 각 요청에 맞는 전략을 수립하고
  • Agent 및 RAG 평가 지표 설계: Agent답변과 RAG의 성능 평가를 위해서 Shannon Entropy와 Simpson Index를 통해 '개인화 지수'와 '향수 중복률'을 평가 지표로 수립하고 이를 기반으로 성능 평가 및 성능 개선을 진행해 나갔습니다.
Infra
  • AWS RDS에 데이터 적재 및 연결: PostgreSQL을 이용하여 RDB를 구성하고 Docker Linux 환경에서 로컬 DB를 호스팅하도록 설계했었고 DB 초기화 및 데이터 자동 적재를 구현하고 AWS RDS를 사용하여 배포하였습니다.
트러블 슈팅 초기 설계 단계에서 설명 가능한 추천을 구현하기 위해 출력 형식을 고정하고자 프롬프트에 Few Shot 방식을 사용하였습니다. 그러나 이 방식은 예외 상황이 발생하거나 검색 결과가 없는 경우에도 에이전트가 고정된 형식에 맞춰 답변을 생성하려다 보니, 데이터베이스에 등록되지 않은 가짜 향수 정보를 만들어내는 할루시네이션 문제가 있었습니다. 그렇다고 에이전트에게 자율성을 더 부여하기에는 프로젝트의 본래 목표인 설명 가능한 추천의 일관성을 유지하기 어려웠습니다. 이를 해결하기 위해 로직상에서 예외 처리와 검색 결과 부재 상황을 명확히 분리하여 각각 적합한 에이전트에게 작업을 할당하는 방식을 채택하였습니다. 또한 에이전트의 자율성 조절을 위해 입력된 정보가 데이터베이스에 존재하는지 여부를 우선적으로 분류하는 단계를 두었습니다. 데이터가 없는 경우에는 별도의 전용 프롬프트를 가진 에이전트가 응대하도록 설계함으로써 할루시네이션을 방지하고 자연스러운 대화 흐름을 확보하며 문제를 해결하였습니다.
기술 스택 PythonDockerAWS EC2PostgresSQLpgvectorLanggraphmodel : gpt 4.1, gpt 5.2
RAG 기반 컴퓨터 비전 유사 특허 검색 챗봇 프로젝트 (Pai) 2025.11~2025.12

컴퓨터 비전과 관련된 사용자의 아이디어를 RAG 검색을 기반으로 한국에 등록된 특허 중에서 유사한 특허를 제시하고 사용자의 아이디어와의 차별점을 제시하고 실제 등록 가능성까지 검토해주는 AI 챗봇 서비스입니다.

역할

데이터

RAG

Agent

Infra

2026 월드컵 순위 예측 ML 프로젝트 2025.9~2025.10
참여인원: 4명

2026년 월드컵의 각 나라별 순위를 이전 월드컵 성적, 예선전 성적, 선수단별 성적 데이터 등을 이용하여 예측하는 ML 모델을 제작하는 프로젝트 입니다. 최종 예측 모델은 매치의 승패를 이진 분류하는 모델을 이용하여 특정 매치업의 승패를 예측하고 이를 시뮬레이션을 통해 평균 수치를 이용하여 순위를 예측하는 모델을 제작하였습니다.

역할

데이터
  • 나라별 월드컵 예선전 성적 데이터 분석 및 전처리: 2014년 부터 약 10년의 데이터를 사용하도록 하였고 예선전 성적과 월드컵 본선 성적의 상관관계를 분석하여 사용방법을 탐색
모델 설계 및 구현
  • XGboost, Random Forest등 ML 모델 적용 및 평가: 처음에는 순위 그 자체를 회귀와 분류로 예측하는 모델을 제작하였으나 20~70%의 성능으로 편차가 크게 나타남. 성적을 명확하게 수치화 하는 것이 아니라 조별리그, 16강, 8강 등의 그룹화 이후 다중 분류 성능을 측정했을 때에는 40%의 안정적이지만 비교적 낮은 수치의 정확도를 보임
  • 매치 예측 모델 설계 및 구현: 특정 매치업의 승패를 예측하는 모델을 제작하여 60% 정도의 정확도를 가진 모델 제작. 무승부에 대한 예측율이 현저하게 떨어지는 것을 확인후 승률 자체를 무승부 확률로 변경하여 처리하기로 결정. 매치 승패 예측의 경우 74%로 성능 향상 이후 해당 매치 승률이 여러 차례의 시뮬레이션 과정을 거치면 전체 결과 예측에 수렴할 것으로 가정하고 시뮬레이션 모델을 제작
트러블 슈팅 초기에는 성적데이터 자체를 이용하여 본선에서의 순위를 회귀로 예측하는 형태로 모델을 설계했지만 월드컵 본선 성적에서 상정하지 못했던 변수가 너무 많고 데이터의 개수가 부족하여 성능 편차가 큰 형태의 모델이 제작되었습니다. 위의 데이터 부족과 상정하지 못한 변수를 최대한 줄이고자 매치 시뮬레이션 형태의 모델로 예측 방식을 변경하였고 결국 시뮬레이션의 횟수가 커진다면 전체 순위 예측의 성능이 매치 예측 모델의 성능에 수렴할 것을 가정하고 모델을 제작하였습니다.
기술 스택 PythonPytorch
배틀그라운드 착륙 위치별 조기 탈락 예측 ML 프로젝트 2025.10~2025.11
참여인원: 5명

배틀그라운드 게임에서 사용자의 평균 스탯 데이터를 이용하여 사용자의 특성을 파악하고 특성에 따라서 지도 상의 특정 지점에 내렸을 때 2분 30초 이내에 사망할 확률을 예측하는 ML 모델을 만드는 프로젝트 입니다.

역할

데이터
  • 인게임 킬로그 데이터 분석 및 전처리: 인게임 킬로그 데이터를 분석하여 특정 시간에 사망한 플레이어와 해당 플레이어의 평균적인 스탯을 매핑
모델 설계 및 구현
  • 1분 30초(모든 플레이어가 지상에 착륙하는 시간) 이내의 사망 지점으로 부터 착륙 지점을 예측하는 ML 모델 설계 및 구현: 착륙지점에 대한 명확한 데이터 부재로 착륙 지점을 예측해야 했는데 게임 시간 1분 30초 이내에 사망한 데이터를 이용하여 극초기에 사망한 좌표는 착륙지점과 거의 유사할 것이다 라고 가설을 수립 후 비교. 해당 킬로그 데이터를 이용하여 지도 상에 표시했을때 사람들이 일반적으로 착륙할 것으로 예상된 지점과 일치여부를 확인. 킬로그 데이터를 이용하여 착륙위치들을 군집화. 92%의 정확도로 착륙위치 예측 성공 (XGBoost모델 이용)
  • 플레이어 스탯 별로 착륙 위치에 따른 2분 30초(플레이어가 착륙이후에 첫번째 교전이 마무리 되는 시간) 이내에 사망할 확률 예측 ML 모델 구현 및 테스트: 사전에 수집된 플레이어의 스탯 (어시스트 수, 다운 횟수, 차량 이동 거리, 도보 이동 거리, 적에게 가한 피해량, 최종적으로 적을 처치한 횟수, 경기 종료 시 팀의 최종 순위)의 평균 데이터와 앞서 구한 예상 착륙 군집을 이용하여 2분 30초 이내에 사망할 확률을 예측. 67%의 정확도를 보임
트러블 슈팅 최초 설계 단계에서는 플레이어의 스탯과 기다 킬로그 데이터를 이용하여 ML 모델 학습을 진행하려 했으나 회의를 통해 배틀그라운드라는 게임 특성상 착륙 위치가 조기사망 예측에 굉장히 중요한 요소라고 판단하였습니다. 하지만 주어진 데이터에는 착륙 좌표에 대한 데이터가 없었기 때문에 게임 진행 극초반 1분 30초(모든 플레이어가 지상에 내려왔을 시점)의 킬로그들을 착륙 지점으로 가정하고 이를 군집화 하여 착륙 지점 군집을 예측 하는 모델을 만들어서 데이터의 부족을 해결하였습니다.
